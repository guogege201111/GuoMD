**Linux 学习计划（为你定制，先打基础再服务当前工作）**

1. 目标（12周）
    - 能独立完成 Linux 开发环境搭建、调试、性能定位。
    - 能用 Linux 工具排查 USB/I2C/PCIe/UDP 链路问题。
    - 能把 Qt/SDK 项目做成“可部署、可观测、可维护”。
2. 每周投入
    - 工作日：每天 45-60 分钟
    - 周末：3-4 小时实战
    - 节奏：学 5 天 + 练 2 天 + 复盘 1 次
3. 12周路线图
4. 第1-2周：Linux 基础与 Shell
    - 文件系统、权限、用户组、进程基础
    - 命令：ls/find/grep/rg/chmod/chown/ps/top/kill/tar
    - 产出：写 2 个 Bash 脚本（日志打包、进程巡检）
5. 第3-4周：开发环境与构建
    - gcc/g++/cmake/make/pkg-config/gdb
    - 动态库、环境变量、ldd、core dump
    - 产出：把你现有一个模块从编译到调试走通并记录步骤
6. 第5-6周：系统与服务管理
    - systemd/journalctl、开机自启、日志轮转
    - ssh/scp/rsync 远程调试流程
    - 产出：把一个 SDK 服务改成 systemd 托管
7. 第7-8周：网络与协议排障（重点）
    - ip/ss/netstat/tcpdump/wireshark/ethtool
    - UDP 延迟、丢包、缓冲区参数（sysctl）
    - 产出：做一份 UDP 链路排障手册（含抓包案例）
8. 第9-10周：性能与实时性（重点）
    - htop/pidstat/perf/strace/ltrace
    - 线程优先级、taskset/chrt、调度基础
    - 产出：对控制链路做一次 P50/P99 延迟分析报告
9. 第11-12周：设备与驱动接口认知
    - /dev、udev、ioctl/mmap 基础
    - 工具：lsusb/i2cdetect/i2cget/lspci/dmesg
    - 产出：完成一次 “设备识别失败到恢复” 的完整排查闭环

10. 学习材料（够用且实战）
    - man + tldr（查命令第一入口）
    - 《鸟哥的 Linux 私房菜（基础学习篇）》
    - 《The Linux Command Line》
    - 《Linux 性能优化实战》（偏工具与排障）
11. 验收标准（3个月后）
    - 你能在 30 分钟内定位“程序卡顿/崩溃/网络异常”主要原因。
    - 你能独立搭建一套“日志 + 抓包 + 性能”诊断流程。
    - 你能和 FPGA/硬件同事用统一证据（日志、时序、抓包）高效联调。




**第1周：命令与文件系统**

1. Day1：目录与路径  
    任务：练 pwd ls -al cd mkdir rm -r  
    验收：在 ~/linux_lab/week1/day1 完成目录创建和删除演练。
2. Day2：文件查看与编辑  
    任务：练 cat less head tail -f wc tee  
    验收：能边看日志边过滤关键字（配合 tail -f | rg）。
3. Day3：搜索能力  
    任务：练 rg find which whereis  
    验收：在你的工程里找到 croestimatework.cpp、包含某函数名的所有文件。
4. Day4：权限基础  
    任务：练 chmod chown groups umask  
    验收：解释清楚 755/644，并给脚本加可执行权限。
5. Day5：进程管理  
    任务：练 ps -ef top/htop kill pkill jobs/bg/fg  
    验收：能启动一个测试进程并在前后台切换、结束它。
6. Day6：脚本1（日志打包）  
    任务：写 pack_logs.sh，自动收集当天日志并 tar.gz。  
    验收：脚本支持一条命令完成打包并输出文件名。
7. Day7：周复盘  
    任务：整理本周命令速查表（20条）。  
    验收：写到 docs/linux/week1-cheatsheet.md。

**第2周：开发与调试基础**

1. Day8：环境变量与Shell配置  
    任务：练 echo $PATH export alias，编辑 ~/.bashrc。  
    验收：配置常用别名（如 ll、gs）。
2. Day9：编译链基础  
    任务：跑通 gcc/g++、make、cmake 基本流程。  
    验收：独立编译一个小C++示例并运行。
3. Day10：GDB入门  
    任务：练断点、单步、栈回溯、变量查看。  
    验收：能定位一个人为制造的空指针崩溃。
4. Day11：动态库与依赖  
    任务：练 ldd nm objdump -T。  
    验收：能看懂一个可执行程序依赖了哪些 .so。
5. Day12：Core dump  
    任务：配置 ulimit -c unlimited，用 gdb 分析 core。  
    验收：输出一次崩溃调用栈记录。
6. Day13：脚本2（进程巡检）  
    任务：写 proc_watch.sh，检查关键进程是否存活。  
    验收：异常时打印告警并返回非0退出码。
7. Day14：周复盘  
    任务：整理“编译失败/运行失败/崩溃”的排查流程图。  
    验收：写到 docs/linux/week2-debug-flow.md。

**第3周：网络与UDP排障（重点）**

1. Day15：网络基本面  
    任务：练 ip a ip r ss -lunpt ping。  
    验收：说明本机IP、路由、监听端口。
2. Day16：UDP收发实验  
    任务：用 nc -u 或 socat 做本机UDP收发。  
    验收：能复现丢包/乱序的基础场景。
3. Day17：抓包基础  
    任务：练 tcpdump -i any udp -nn 过滤表达式。  
    验收：抓到指定端口包并保存 .pcap。
4. Day18：Wireshark分析  
    任务：打开 .pcap 看时序、长度、重传/丢失迹象。  
    验收：写3条观察结论。
5. Day19：网络参数调优认知  
    任务：看 sysctl：net.core.rmem_max wmem_max 等。  
    验收：知道“收发缓冲区”对UDP的影响。
6. Day20：项目化输出  
    任务：写《UDP链路排障SOP》初版。  
    验收：包含“现象-命令-证据-结论”四段。
7. Day21：周复盘  
    任务：拿你当前项目一个端口做真实抓包。  
    验收：提交1份你自己的排障案例。

**第4周：系统服务与性能定位**

1. Day22：systemd基础  
    任务：学习 systemctl，写一个最小 .service。  
    验收：服务能 start/stop/restart。
2. Day23：日志系统  
    任务：练 journalctl -u -f --since。  
    验收：按时间窗口定位一次错误日志。
3. Day24：远程调试  
    任务：练 ssh scp rsync。  
    验收：完成一次远程日志拉取。
4. Day25：性能工具入门  
    任务：练 pidstat strace perf stat。  
    验收：对一个进程输出CPU/系统调用热点结论。
5. Day26：调度与实时性基础  
    任务：练 taskset chrt，理解优先级影响。  
    验收：完成一次线程亲和性实验。
6. Day27：小项目整合  
    任务：把“日志收集+进程巡检+端口检查”整成一个脚本包。  
    验收：一键执行并生成报告。
7. Day28：月度验收  
    任务：做一次完整演练：启动服务->异常->定位->修复->复盘。  
    验收：形成 docs/linux/month1-report.md
