1、核心概念
```
1. PID：进程号。
2. PPID：父进程号。
3. 前台进程：占用当前终端。后台进程：终端可继续输入命令。
4. 信号：TERM(15)优雅退出，KILL(9)强制终止。
```

3、PS用法
一张快照
```
// 查看SkyOCT的pid号
ps aux |grep SkyOCT

a：显示终端上的所有进程，包括其他用户的进程。
  
u：显示面向用户的格式（包含 %CPU、%MEM、VSZ、RSS 等）。
  
x：显示没有控制终端的进程。


// 按 CPU 使用率排序显示前几行
ps aux --sort=-%cpu |head

// 看进程快照 -e：显示所有进程，-f：full format，用更完整的格式显示
ps -ef

输出常见列：

- UID：用户
- PID：进程号
- PPID：父进程号
- C：CPU 使用相关的简略值
- STIME：启动时间
- TTY：终端
- TIME：累计 CPU 时间
- CMD：命令行
  
```

4、top用法
实时刷新
```
**top 顶部常见信息**

- load average：系统负载
- Tasks：任务数
- %Cpu(s)：CPU 使用情况
- MiB Mem：内存使用情况
- MiB Swap：交换分区情况
  
  你会看到两部分：

1. 顶部：系统总体状态
2. 下方：进程列表

**top 顶部常见信息**

- load average：系统负载
- Tasks：任务数
- %Cpu(s)：CPU 使用情况
- MiB Mem：内存使用情况
- MiB Swap：交换分区情况

**先理解 load average**

很多新手把它当 CPU 使用率，这是错的。

load average 表示一段时间内“处于可运行状态或不可中断睡眠状态”的平均任务数。  
常看 3 个值：

- 1 分钟
- 5 分钟
- 15 分钟

如果你是 8 核机器：

- load 在 8 左右，通常算满负荷附近
- 远超 8，说明排队严重或有大量阻塞

但还要结合 CPU 和 D 状态一起判断。

例如：

- load 很高，CPU 不高，很多进程是 D  
    这很可能是 IO/驱动阻塞，不是纯 CPU 忙

这在驱动/系统优化里非常关键。

**top 进程区常见列**

- PID
- USER
- PR：优先级
- NI：nice 值
- VIRT：虚拟内存
- RES：常驻内存
- SHR：共享内存
- S：状态
- %CPU
- %MEM
- TIME+：累计 CPU 时间
- COMMAND

**top 里最实用的交互键**

- P：按 CPU 排序
- M：按内存排序
- T：按运行时间排序
- 1：展开每个 CPU 核的统计
- H：显示线程
- k：杀进程
- r：调整 nice 值
- q：退出

如果你要排查某个进程的线程谁最忙，非常实用：

`top -H -p 1234`
```
