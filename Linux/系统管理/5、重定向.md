
在 Shell 里，大多数命令启动后默认会连着 3 个“数据通道”：

- 0：标准输入，stdin
- 1：标准输出，stdout
- 2：标准错误，stderr
-
1、> ：把标准输出重定向到文件，覆盖写
```
dmesg > dmesg.log

```

2、>>：把标准输出追加到文件末尾，不覆盖
```
dmesg >> dmesg.log
```

3、2>：把标准错误重定向到文件，覆盖写
```
ls no_such_dir 2> err.log

// 如果模块加载失败，错误信息就会进文件。
insmod mydrv.ko 2> insmod_err.log


```

4、2>&1：让标准错误跟标准输出去同一个地方
```
// 编译的正常输出写到 build.log，编译报错也写到 build.log
make > build.log 2>&1

```

5、`<` – 从文件读取输入
这是最基本的输入重定向。将指定文件的内容作为命令的标准输入。
```
# 从 input.txt 读取，输出到 output.txt
./program < input.txt > output.txt
```

6、`<<` – Here Document（内嵌文档）
用于在脚本或命令行中直接提供**多行**的输入文本，直到遇到一个自定义的**分隔符**为止。所有行（包括换行符）都会被当作标准输入发送给命令。
```
基本格式：

命令 << 分隔符
文本行1
文本行2
...
分隔符

- **分隔符**：可以是任意字符串（如 `EOF`、`END`、`STOP`）。必须单独占一行，且该行不能有其他字符（包括空格）。通常使用 `EOF` 或 `END`。
    
- 分隔符前后的文本会原样传递，但会进行**变量扩展**、**命令替换**等 Shell 解析（除非将分隔符用引号包围，见下文）。
  
  
  # 使用 cat 直接输出多行文本到屏幕
cat << EOF
这是第1行
这是第2行，当前日期是 $(date)
EOF

```

7、 `<<<` – Here String（内嵌字符串）
这是 Here Document 的单行版本。将**一个字符串**（通常带引号）作为标准输入传递给命令。字符串末尾会自动添加一个换行符（除非特别指定）
```
# 统计字符串中的单词数
wc -w <<< "Hello world"   # 输出 2

# 将字符串转为大写
tr 'a-z' 'A-Z' <<< "hello"   # 输出 HELLO

# 读取字符串中的字段
read a b c <<< "A B C"
echo $b   # 输出 B
```
